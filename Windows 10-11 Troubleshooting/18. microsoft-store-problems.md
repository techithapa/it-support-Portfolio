# Overview

The Microsoft Store is used to install and update applications, including enterprise apps distributed through Intune or the Windows Package Manager. When the Store fails, users may be unable to download apps, update existing apps, or access Store services.

Issues may arise from corrupted Store cache, damaged app packages, network restrictions, incorrect account settings, Windows Update failures, or system file corruption.

This guide provides a structured troubleshooting process for diagnosing and resolving Microsoft Store issues in Windows 10 and Windows 11.

---

# Symptoms

Users may report:

- Microsoft Store not opening  
- Store crashes immediately  
- Apps stuck on “Pending” or “Downloading”  
- Error codes (0x80070422, 0x80131500, 0x80072EFD)  
- Cannot sign in to Store  
- Blank Store window  
- Store apps not updating  
- “Something went wrong” messages  
- Slow or unresponsive Store

# Step 1 – Identify the Issue

Determine:

- Does the Store open?  
- Does the Store crash?  
- Are apps stuck downloading?  
- Does the Store fail only on VPN?  
- Does the Store fail for all users?  

Record:

- Error codes  
- Network type  
- Account type  

---

# Step 2 – Check Internet & Account Connectivity

Verify internet:

```cmd
ping www.microsoft.com
```

Check Store services:

```powershell
Test-NetConnection -ComputerName storeedgefd.dsx.mp.microsoft.com -Port 443
```

Sign out and sign back into:

```
Microsoft Store
Microsoft Account / Entra ID
```

---

# Step 3 – Reset Microsoft Store Cache

Run:

```cmd
wsreset.exe
```

This clears corrupted Store cache files.

---

# Step 4 – Reinstall Microsoft Store

PowerShell (Admin):

```powershell
Get-AppxPackage -allusers Microsoft.WindowsStore | Remove-AppxPackage
```

Reinstall:

```powershell
Get-AppxPackage -allusers Microsoft.WindowsStore | Foreach {
    Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppxManifest.xml"
}
```

---

# Step 5 – Repair/Reset Microsoft Store App

Navigate to:

```
Settings
Apps
Installed Apps
Microsoft Store
Advanced Options
```

Select:

- **Repair**  
- **Reset**  

---

# Step 6 – Check Windows Update & Licensing Services

Ensure services are running:

```cmd
sc query wuauserv
sc query clipsvc
sc query licensemanager
```

Restart services:

```cmd
net stop clipsvc
net start clipsvc
```

---

# Step 7 – Run System File Checker (SFC)

```cmd
sfc /scannow
```

Repairs corrupted system files affecting Store functionality.

Restart Windows.

---

# Step 8 – Repair Windows Image (DISM)

If SFC reports unrepairable files:

```cmd
DISM /Online /Cleanup-Image /RestoreHealth
```

Restart Windows.

---

# Step 9 – Check Date, Time & Region Settings

Incorrect settings can break Store authentication.

Navigate to:

```
Settings
Time & Language
Date & Time
```

Enable:

- Set time automatically  
- Set time zone automatically  

Region:

```
Settings
Time & Language
Language & Region
```

Set correct region (Australia for Sunshine Coast).

---

# Step 10 – Review Event Viewer Logs

Open:

```
eventvwr.msc
```

Navigate to:

```
Applications and Services Logs
Microsoft
Windows
Store
```

Look for:

- Licensing errors  
- Store crashes  
- Network failures  

---

# Step 11 – PowerShell Store Diagnostics

List Store apps:

```powershell
Get-AppxPackage Microsoft.WindowsStore
```

Check Store dependencies:

```powershell
Get-AppxPackage | Where-Object {$_.Name -like "*Store*"}
```

Check recent errors:

```powershell
Get-EventLog Application -Newest 20
```

# Step 12 – Advanced Troubleshooting

## Reset Windows Update Components

```cmd
net stop wuauserv
net stop bits
rd /s /q C:\Windows\SoftwareDistribution
net start wuauserv
net start bits
```

---

## Clear Delivery Optimization Cache

```powershell
Clear-DeliveryOptimizationCache
```

---

## Re-register all UWP apps

```powershell
Get-AppxPackage -AllUsers | Foreach {
    Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppxManifest.xml"
}
```

---

## Test with New User Profile

If Store works under a new profile → original profile corrupted.

# Verification

Confirm:

- Store opens normally  
- Apps download and update  
- No Store crashes  
- No new Store errors in Event Viewer  
- Licensing service functioning  
- Network connectivity stable  

---

# Common Error Messages

| Error | Possible Cause |
|-------|----------------|
| 0x80070422 | Windows Update disabled |
| 0x80131500 | Store cache corruption |
| 0x80072EFD | Network/proxy issue |
| Blank Store window | UWP corruption |
| Store not opening | Damaged Store package |

---

# Preventive Best Practices

- Keep Windows updated  
- Avoid aggressive proxy filtering  
- Maintain correct date/time settings  
- Restart Windows periodically  
- Avoid deleting UWP system files  
- Ensure licensing services remain enabled  

---

# Escalation Criteria

Escalate when:

- Store fails across multiple devices  
- Licensing service repeatedly fails  
- Windows image cannot be repaired  
- UWP apps fail system-wide  
- Proxy or firewall restrictions require admin review  
- Windows reinstallation is being considered
