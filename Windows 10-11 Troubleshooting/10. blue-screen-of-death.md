# Overview

Blue Screen of Death (BSOD) errors occur when Windows encounters a critical system failure that it cannot recover from. These failures may be caused by driver conflicts, hardware faults, memory corruption, system file damage, or kernel-level issues.

This guide provides a structured troubleshooting process for diagnosing and resolving BSOD errors in Windows 10 and Windows 11.

---

# Symptoms

Users may report:

- Sudden system crash with blue screen  
- Automatic restart after crash  
- Repeated BSOD loops  
- System freezing before BSOD  
- BSOD after Windows update  
- BSOD during heavy workloads (gaming, rendering, etc.)  
- Minidump files generated in `C:\Windows\Minidump`

# Common Causes

## Drivers

- Outdated or corrupted drivers  
- Faulty GPU, network, or chipset drivers  
- Incorrect driver installation  

---

## Hardware

- Faulty RAM  
- Failing SSD/HDD  
- Overheating CPU/GPU  
- Power supply instability  

---

## Windows

- Corrupted system files  
- Damaged Windows image  
- Failed updates  
- Kernel-level failures  

---

## Software

- Antivirus conflicts  
- Virtualization software  
- Overclocking utilities

# Step 1 – Identify the BSOD Error

Record:

- Stop code (e.g., `CRITICAL_PROCESS_DIED`)  
- Error message  
- Time of crash  
- Actions performed before crash  

Common BSOD stop codes are listed later in this document.

---

# Step 2 – Collect Crash Logs

Minidump files:

```
C:\Windows\Minidump
```

System crash logs:

```powershell
Get-EventLog -LogName System -Newest 50
```

Reliability Monitor:

```
perfmon /rel
```

---

# Step 3 – Run Memory Diagnostics

Memory issues cause many BSOD errors.

Run:

```cmd
mdsched.exe
```

Restart and allow test to complete.

---

# Step 4 – Check System File Integrity

Run SFC:

```cmd
sfc /scannow
```

Restart Windows.

---

# Step 5 – Repair Windows Image (DISM)

If SFC reports unrepairable files:

```cmd
DISM /Online /Cleanup-Image /RestoreHealth
```

Restart Windows.

---

# Step 6 – Update/Roll Back Drivers

Open:

```
devmgmt.msc
```

Update drivers:

```cmd
pnputil /scan-devices
```

Roll back problematic drivers:

```
Device Manager → Properties → Driver → Roll Back Driver
```

---

# Step 7 – Check Hardware Health

## Disk Health

```cmd
wmic diskdrive get status
```

Expected:

```
OK
```

## CHKDSK

```cmd
chkdsk C: /f /r
```

## Temperature Monitoring

Check overheating using OEM tools.

---

# Step 8 – Analyze Minidump Files

Use WinDbg:

```cmd
windbg.exe
```

Load dump file:

```
File → Open Crash Dump
```

Run analysis:

```cmd
!analyze -v
```

Or use BlueScreenView for simplified analysis.

---

# Step 9 – Review Event Viewer Logs

Open:

```
eventvwr.msc
```

Navigate to:

```
Windows Logs
System
```

Look for:

- Kernel errors  
- Driver failures  
- Disk errors  
- Memory errors  

---

# Step 10 – Advanced Troubleshooting

## Disable Overclocking

Reset BIOS to default settings.

---

Uninstall:

- Third‑party antivirus  
- Virtualization tools  
- Overclocking utilities

## Test with New User Profile

If BSOD stops → profile corruption.

# Verification

Confirm:

- No BSOD after extended use  
- No new errors in Event Viewer  
- System stable under load  
- Drivers updated and functioning

# Common BSOD Error Codes

| Error Code | Meaning |
|------------|---------|
| CRITICAL_PROCESS_DIED | System process failure |
| MEMORY_MANAGEMENT | RAM corruption |
| IRQL_NOT_LESS_OR_EQUAL | Driver conflict |
| PAGE_FAULT_IN_NONPAGED_AREA | Invalid memory access |
| SYSTEM_SERVICE_EXCEPTION | Driver/service failure |
| KERNEL_SECURITY_CHECK_FAILURE | Kernel integrity violation |
| BAD_POOL_CALLER | Memory pool corruption |
| DPC_WATCHDOG_VIOLATION | Hardware timeout |
| UNEXPECTED_STORE_EXCEPTION | Storage subsystem failure |
| VIDEO_TDR_FAILURE | GPU timeout |

---

# Preventive Best Practices

- Keep drivers updated  
- Maintain Windows updates  
- Avoid unnecessary overclocking  
- Monitor hardware temperatures  
- Use high‑quality hardware components  
- Restart Windows periodically  

---

# Escalation Criteria

Escalate when:

- BSOD persists after repairs  
- Hardware failure suspected  
- Minidump analysis indicates kernel corruption  
- Windows image cannot be repaired  
- Enterprise drivers conflict  
- Windows reinstallation required
