# Overview

Windows Defender (Microsoft Defender Antivirus) provides built‑in protection against malware, ransomware, and other threats. When Defender fails, users may be unable to run scans, update definitions, or access Windows Security features.

Issues may arise from disabled services, corrupted Defender components, conflicting third‑party antivirus software, damaged system files, or policy restrictions.

This guide provides a structured troubleshooting process for diagnosing and resolving Windows Defender issues in Windows 10 and Windows 11.

---

# Symptoms

Users may report:

- Windows Defender not opening  
- “Threat service has stopped”  
- Unable to run scans  
- Virus & threat protection missing  
- Defender updates failing  
- Real‑time protection disabled  
- “Your IT administrator has limited access”  
- Windows Security blank or unresponsive  
- Defender quarantines not accessible

# Step 1 – Identify the Issue

Determine:

- Does Windows Security open?  
- Does Defender show “Threat service stopped”?  
- Are updates failing?  
- Is Defender disabled by policy?  
- Is third‑party antivirus installed?  

Record:

- Error messages  
- Policy restrictions  
- Update status  

---

# Step 2 – Check Windows Defender Service Status

Check services:

```cmd
sc query WinDefend
sc query SecurityHealthService
sc query wscsvc
```

Expected:

```
STATE: RUNNING
```

Restart services:

```cmd
net start WinDefend
net start SecurityHealthService
net start wscsvc
```

---

# Step 3 – Verify Windows Security Components

Check Defender health:

```powershell
Get-MpComputerStatus
```

Check real‑time protection:

```powershell
Get-MpPreference | Select-Object DisableRealtimeMonitoring
```

Expected:

```
DisableRealtimeMonitoring : False
```

---

# Step 4 – Update Defender Signatures

Update definitions:

```powershell
Update-MpSignature
```

Check update status:

```powershell
Get-MpComputerStatus | Select AntivirusSignatureVersion
```

---

# Step 5 – Run Windows Defender Health Check

Run quick scan:

```powershell
Start-MpScan -ScanType QuickScan
```

Run full scan:

```powershell
Start-MpScan -ScanType FullScan
```

Check protection status:

```powershell
Get-MpThreat
```

---

# Step 6 – Reset Windows Security App

Reset Windows Security:

```powershell
Get-AppxPackage Microsoft.SecHealthUI | Reset-AppxPackage
```

Reinstall Windows Security:

```powershell
Add-AppxPackage -Register "C:\Windows\SystemApps\Microsoft.SecHealthUI_cw5n1h2txyewy\AppxManifest.xml" -DisableDevelopmentMode
```

---

# Step 7 – Run System File Checker (SFC)

```cmd
sfc /scannow
```

Repairs corrupted Defender components.

Restart Windows.

---

# Step 8 – Repair Windows Image (DISM)

If SFC reports unrepairable files:

```cmd
DISM /Online /Cleanup-Image /RestoreHealth
```

Restart Windows.

---

# Step 9 – Check Group Policy & MDM Restrictions

Open:

```
gpedit.msc
```

Navigate to:

```
Computer Configuration
Administrative Templates
Windows Components
Microsoft Defender Antivirus
```

Check:

- Turn off Microsoft Defender Antivirus  
- Allow real‑time protection  
- Cloud‑delivered protection  

Intune:

```
Endpoint Security → Antivirus
```

Check:

- Tamper protection  
- Real‑time protection  
- Cloud protection


# Step 11 – PowerShell Defender Diagnostics

List Defender preferences:

```powershell
Get-MpPreference
```

Check Defender services:

```powershell
Get-Service WinDefend
```

Check recent Defender events:

```powershell
Get-EventLog -LogName System -Newest 20
```

# Step 12 – Advanced Troubleshooting

## Remove Conflicting Antivirus

Uninstall third‑party antivirus:

```cmd
appwiz.cpl
```

Restart Windows.

---

## Reset Defender Registry Keys

```powershell
Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows Defender" -Name "DisableAntiSpyware" -Value 0
```

---

## Re-register Defender Components

```powershell
Get-AppxPackage -AllUsers Microsoft.Windows.SecHealthUI | Foreach {
    Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppxManifest.xml"
}
```

---

## Test with New User Profile

If Defender works under a new profile → original profile corrupted.

# Verification

Confirm:

- Windows Security opens normally  
- Defender real‑time protection enabled  
- Signature updates succeed  
- No Defender service failures  
- No new Defender errors in Event Viewer

# Common Error Messages

| Error | Possible Cause |
|-------|----------------|
| “Threat service has stopped” | Service failure |
| 0x80070422 | Defender disabled |
| 0x800106ba | Corrupted Defender components |
| “Your IT administrator has limited access” | Policy restriction |
| Blank Windows Security | UWP corruption |

---

# Preventive Best Practices

- Keep Windows updated  
- Avoid installing multiple antivirus products  
- Maintain correct Defender policies  
- Enable Tamper Protection  
- Restart Windows periodically  
- Monitor Defender logs  

---

# Escalation Criteria

Escalate when:

- Defender fails across multiple devices  
- Policies conflict with enterprise security baselines  
- Windows image cannot be repaired  
- Defender services repeatedly stop  
- Third‑party antivirus removal fails  
- Windows reinstallation is being considered
