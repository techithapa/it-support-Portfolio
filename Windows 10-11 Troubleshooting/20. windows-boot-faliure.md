# Overview

Windows boot failures prevent the operating system from loading, leaving users stuck at a black screen, spinning dots, or repeated restart loops. Boot failures may be caused by corrupted boot files, damaged system components, failed updates, hardware issues, or misconfigured BIOS/UEFI settings.

This guide provides a structured troubleshooting process for diagnosing and resolving Windows boot failures in Windows 10 and Windows 11.

---

# Symptoms

Users may report:

- Windows stuck on spinning dots  
- Black screen after BIOS  
- “Automatic Repair couldn’t repair your PC”  
- “Boot device not found”  
- Repeated restart loop  
- Blue screen before login  
- Windows logo appears then freezes  
- “Inaccessible Boot Device”  
- WinRE loads automatically

# Step 1 – Identify the Issue

Determine:

- Does BIOS detect the drive?  
- Does Windows attempt to load?  
- Does WinRE appear automatically?  
- Did issue begin after updates?  
- Is BitLocker enabled?  

Record:

- Error messages  
- Boot stage  
- Hardware changes  

---

# Step 2 – Check Hardware & Power Integrity

Verify:

- SSD/HDD connected securely  
- No damaged cables  
- Laptop battery not failing  
- External devices disconnected  

Test drive in BIOS:

```
Information → Storage
```

If drive missing → hardware failure likely.

---

# Step 3 – Access Windows Recovery Environment (WinRE)

Enter WinRE:

```
Power on → Hold Shift → Restart
```

or automatic after failed boot.

Navigate to:

```
Troubleshoot
Advanced Options
```

---

# Step 4 – Run Startup Repair

Open:

```
Troubleshoot
Advanced Options
Startup Repair
```

Startup Repair attempts to fix:

- Bootloader  
- Registry  
- System files  
- Driver conflicts  

---

# Step 5 – Repair Boot Records (MBR/EFI)

Open Command Prompt in WinRE.

### MBR Systems

```cmd
bootrec /fixmbr
bootrec /fixboot
bootrec /scanos
bootrec /rebuildbcd
```

### UEFI Systems

Rebuild EFI boot files:

```cmd
bcdboot C:\Windows /l en-us /s EFI /f ALL
```

---

# Step 6 – Check Disk Health

Run CHKDSK:

```cmd
chkdsk C: /f /r
```

If disk shows bad sectors → replace drive.

---

# Step 7 – Run SFC & DISM from WinRE

### SFC Offline Scan

```cmd
sfc /scannow /offbootdir=C:\ /offwindir=C:\Windows
```

### DISM Offline Repair

```cmd
DISM /Image:C:\ /Cleanup-Image /RestoreHealth
```

---

# Step 8 – Restore Registry Hives

Navigate to:

```
C:\Windows\System32\config
```

Backup corrupted hives:

```cmd
ren DEFAULT DEFAULT.bak
ren SAM SAM.bak
ren SECURITY SECURITY.bak
ren SOFTWARE SOFTWARE.bak
ren SYSTEM SYSTEM.bak
```

Restore from RegBack:

```cmd
copy C:\Windows\System32\config\RegBack\* C:\Windows\System32\config\
```

---

# Step 9 – Roll Back Windows Updates

Navigate to:

```
Troubleshoot
Advanced Options
Uninstall Updates
```

Choose:

- Latest quality update  
- Latest feature update  

---

# Step 10 – Use System Restore

Open:

```
Troubleshoot
Advanced Options
System Restore
```

Select restore point before failure.

---

# Step 11 – Review Boot Logs

Enable boot logging:

```cmd
bcdedit /set {default} bootlog Yes
```

Logs stored in:

```
C:\Windows\ntbtlog.txt
```

Review for:

- Driver failures  
- Missing files  
- Hardware issues  

---

# Step 12 – Advanced Troubleshooting

## Disable Driver Signature Enforcement

```cmd
bcdedit /set nointegritychecks on
```

---

## Disable Early Launch Anti-Malware

```
Startup Settings → Disable ELAM
```

---

## Rebuild Entire Boot Configuration

```cmd
bcdedit /export C:\bcdbackup
attrib c:\boot\bcd -h -r -s
ren c:\boot\bcd bcd.old
bootrec /rebuildbcd
```

---

## Test RAM

```cmd
mdsched.exe
```

---

## Replace Drive (if SMART failure)

SMART check:

```cmd
wmic diskdrive get status
```

Expected:

```
OK
```

# Verification

Confirm:

- Windows boots normally  
- No restart loops  
- No bootloader errors  
- No BitLocker recovery prompts  
- No new boot errors in Event Viewer

# Common Error Messages

| Error | Possible Cause |
|-------|----------------|
| “Inaccessible Boot Device” | Driver or disk failure |
| “Automatic Repair couldn’t repair your PC” | Bootloader corruption |
| 0xc000000f | Missing boot files |
| 0xc0000225 | Damaged BCD |
| Black screen | GPU or boot corruption |
| Spinning dots | Driver or update failure |

---

# Preventive Best Practices

- Keep Windows updated  
- Maintain healthy SSD/HDD  
- Avoid forced shutdowns  
- Ensure BIOS/UEFI settings remain consistent  
- Back up registry and system images  
- Document hardware changes  
- Enable BitLocker recovery key backup  

---

# Escalation Criteria

Escalate when:

- Drive not detected in BIOS  
- Repeated boot failures after repairs  
- Windows image cannot be repaired  
- Registry corruption persists  
- Hardware failure suspected  
- Enterprise boot policies conflict  
- Windows reinstallation is required
