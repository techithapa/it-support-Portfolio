# Overview

USB devices such as flash drives, keyboards, mice, printers, and external hard drives may fail to be recognized by Windows 10 or Windows 11. This can occur due to driver corruption, power management issues, faulty USB ports, damaged cables, or system-level corruption.

This guide provides a structured troubleshooting process for diagnosing and resolving USB recognition issues in enterprise environments.

---

# Symptoms

Users may report:

- “USB device not recognized”  
- Device not appearing in File Explorer  
- Device missing from Device Manager  
- USB ports not responding  
- Intermittent USB connectivity  
- USB device disconnects randomly  
- External HDD not spinning or powering  
- USB keyboard/mouse not detected  
- USB device works on other computers

# Common Causes

## Windows

- Corrupted USB drivers  
- Disabled USB controllers  
- Power management conflicts  
- System file corruption  

---

## Hardware

- Faulty USB ports  
- Damaged cables  
- Insufficient power for external drives  
- Docking station issues  

---

## User Profile

- Corrupted device installation history  
- Incorrect permissions  

---

## Third-Party Software

- Security software blocking USB  
- USB encryption tools  
- Virtualization software interfering with USB passthrough

# Step 1 – Identify the Issue

Determine:

- Does the device power on?  
- Does the device work on another PC?  
- Does the port work with other devices?  
- Did the issue begin after updates?  

Record:

- Device type  
- Port type (USB-A / USB-C)  
- Error messages  

---

# Step 2 – Check Physical USB Connections

Verify:

- Device firmly connected  
- Try different USB ports  
- Try different cable  
- Try connecting directly (avoid docking station)  
- Test device on another computer  

If device works elsewhere → Windows or driver issue.

---

# Step 3 – Verify Device Detection

Open:

```
devmgmt.msc
```

Look for:

- Unknown Device  
- USB Mass Storage Device  
- Yellow exclamation marks  

Check Disk Management:

```
diskmgmt.msc
```

If device appears without a drive letter:

Assign one manually.

---

# Step 4 – Restart USB Controllers

Restart USB controllers:

```cmd
devmgmt.msc
```

Expand:

```
Universal Serial Bus controllers
```

Right-click each:

```
Uninstall device
```

Restart Windows.

Or use command-line:

```powershell
Get-PnpDevice -Class USB | Disable-PnpDevice -Confirm:$false
Get-PnpDevice -Class USB | Enable-PnpDevice -Confirm:$false
```

---

# Step 5 – Update/Reinstall USB Drivers

Rescan devices:

```cmd
pnputil /scan-devices
```

Remove corrupted drivers:

```cmd
pnputil /delete-driver oem*.inf /uninstall
```

Install latest OEM chipset drivers.

---

# Step 6 – Disable USB Selective Suspend

Open:

```
Control Panel
Power Options
Change plan settings
Change advanced power settings
USB settings
USB selective suspend setting
Disabled
```

This prevents Windows from powering down USB devices.

---

# Step 7 – Run System File Checker (SFC)

```cmd
sfc /scannow
```

Repairs corrupted system files affecting USB functionality.

Restart Windows.

---

# Step 8 – Repair Windows Image (DISM)

If SFC reports unrepairable files:

```cmd
DISM /Online /Cleanup-Image /RestoreHealth
```

Restart Windows.

---

# Step 9 – Check Power Management Settings

Disable power saving on USB hubs:

Open:

```
devmgmt.msc
```

Expand:

```
Universal Serial Bus controllers
```

For each USB Root Hub:

```
Properties
Power Management
Uncheck "Allow the computer to turn off this device to save power"
```

---

# Step 10 – Review Event Viewer Logs

Open:

```
eventvwr.msc
```

Navigate to:

```
Windows Logs
System
```

Look for:

- USB driver failures  
- Device enumeration errors  
- Power management issues  

---

# Step 11 – PowerShell USB Diagnostics

List USB devices:

```powershell
Get-PnpDevice -Class USB
```

Check device status:

```powershell
Get-PnpDevice -Class USB | Select Status, FriendlyName
```

Check recent USB errors:

```powershell
Get-EventLog System -Newest 20
```

---

# Step 12 – Advanced Troubleshooting

## Reset USB Stack

```cmd
net stop usbccgp
net start usbccgp
```

---

## Remove USB Installation History

```powershell
Get-PnpDevice -Class USB | Remove-PnpDevice -Confirm:$false
```

---

## Test with New User Profile

If USB works under a new profile → original profile corrupted.

---

## Check BIOS/UEFI Settings

Verify:

- USB ports enabled  
- Legacy USB support enabled  
- USB-C alternate mode settings  

---

## Docking Station Firmware Update

Update docking station firmware if USB-C devices fail.

---

# Verification

Confirm:

- USB device recognized  
- Device appears in Device Manager  
- Device accessible in File Explorer  
- No new USB errors in Event Viewer  
- Device works across multiple ports

# Common Error Messages

| Error | Possible Cause |
|-------|----------------|
| “USB device not recognized” | Driver corruption |
| “Unknown USB device” | Enumeration failure |
| “Device descriptor request failed” | Hardware issue |
| Device not appearing | Power management or port failure |
| Device disconnects randomly | Faulty cable or selective suspend |

---

# Preventive Best Practices

- Keep chipset and USB drivers updated  
- Avoid low-quality USB cables  
- Maintain Windows updates  
- Disable USB selective suspend for critical devices  
- Use powered USB hubs for external HDDs  
- Restart Windows periodically  

---

# Escalation Criteria

Escalate when:

- USB fails across multiple devices  
- Ports physically damaged  
- Drivers repeatedly corrupt  
- Windows image cannot be repaired  
- Docking station hardware failure suspected  
- BIOS/UEFI issues require vendor support
