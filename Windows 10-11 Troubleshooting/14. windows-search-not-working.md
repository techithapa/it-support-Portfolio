# Overview

Windows Search provides fast indexing and retrieval of files, applications, settings, and emails. When Windows Search stops working, users may be unable to find files, launch applications, or use the Start Menu search bar.

Search failures may be caused by corrupted indexes, disabled services, damaged system files, profile corruption, Group Policy restrictions, or Windows component issues.

This guide provides a structured troubleshooting process for diagnosing and resolving Windows Search issues in Windows 10 and Windows 11.

---

# Symptoms

Users may report:

- Search bar not responding  
- Search results not loading  
- Blank search window  
- Search crashes or freezes  
- Search indexing stuck  
- Cannot find files or apps  
- Search works slowly  
- Start Menu search not opening  
- High CPU usage from SearchIndexer.exe

# Common Causes

## Windows

- Corrupted search index  
- Damaged Cortana/Search components  
- Disabled Windows Search service  
- Windows update issues  

---

## Storage

- Slow HDD performance  
- File system corruption  
- Indexing large or inaccessible folders  

---

## User Profile

- Corrupted profile  
- Damaged registry keys  
- Incorrect permissions  

---

## Group Policy

- Search disabled by policy  
- Cortana disabled  
- Indexing restrictions

# Step 1 – Identify the Issue

Determine:

- Does search open?  
- Does search show results?  
- Does indexing run?  
- Does search fail for all users?  
- Did issue begin after updates?  

Record:

- Error messages  
- Time of occurrence  
- Affected folders  

---

# Step 2 – Restart Windows Search Services

Restart Search service:

```cmd
net stop wsearch
net start wsearch
```

Check service status:

```cmd
sc query wsearch
```

Expected:

```
STATE: RUNNING
```

---

# Step 3 – Rebuild Search Index

Open:

```
Control Panel
Indexing Options
Advanced
Rebuild
```

Command-line rebuild:

```powershell
Get-Service wsearch | Restart-Service
```

---

# Step 4 – Check Windows Search Settings

Navigate to:

```
Settings
Privacy & Security
Searching Windows
```

Verify:

- Indexing enabled  
- Correct folders selected  
- Enhanced search mode (optional)  

---

# Step 5 – Run System File Checker (SFC)

```cmd
sfc /scannow
```

Repairs corrupted system files affecting search.

Restart Windows.

---

# Step 6 – Repair Windows Image (DISM)

If SFC reports unrepairable files:

```cmd
DISM /Online /Cleanup-Image /RestoreHealth
```

Restart Windows.

---

# Step 7 – Reset Cortana/Search Components

Reset Search UI:

```powershell
Get-AppxPackage Microsoft.Windows.Search | Reset-AppxPackage
```

Reset Cortana:

```powershell
Get-AppxPackage Microsoft.549981C3F5F10 | Reset-AppxPackage
```

---

# Step 8 – Check Group Policy Restrictions

Open:

```
gpedit.msc
```

Navigate to:

```
Computer Configuration
Administrative Templates
Windows Components
Search
```

Check:

- Allow Cortana  
- Do not allow search  
- Disable indexing encrypted items  
- Prevent indexing certain paths  

Apply policy:

```cmd
gpupdate /force
```

---

# Step 9 – Validate Search Registry Keys

Check Search key:

```powershell
Get-Item "HKLM:\SOFTWARE\Microsoft\Windows Search"
```

Reset key:

```powershell
Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows Search" -Name "SetupCompletedSuccessfully" -Value 1
```

---

# Step 10 – Review Event Viewer Logs

Open:

```
eventvwr.msc
```

Navigate to:

```
Applications and Services Logs
Microsoft
Windows
Search
```

Look for:

- Index corruption  
- Search service crashes  
- Permission issues  

---

# Step 11 – PowerShell Search Diagnostics

Check index status:

```powershell
Get-WmiObject -Class MSFT_SearchIndexer
```

Check search service:

```powershell
Get-Service wsearch
```

Check recent errors:

```powershell
Get-EventLog Application -Newest 20
```

---

# Step 12 – Advanced Troubleshooting

## Create a New User Profile

If search works under a new profile, original profile is corrupted.

---

## Reset Search Database

```cmd
net stop wsearch
rd /s /q "C:\ProgramData\Microsoft\Search\Data\Applications\Windows"
net start wsearch
```

---

## Check File System Health

```cmd
chkdsk C: /f
```

---

## Disable Bing Search Integration (optional)

```powershell
reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Search" /v BingSearchEnabled /t REG_DWORD /d 0 /f
```

# Verification

Confirm:

- Search opens instantly  
- Search results appear  
- Indexing completes successfully  
- No search crashes  
- Event Viewer shows no new search errors  
- Start Menu search works normally

# Common Error Messages

| Error | Possible Cause |
|-------|----------------|
| SearchUI.exe crash | Corrupted Cortana/Search components |
| Blank search window | Damaged registry keys |
| Indexing stopped | Service disabled |
| Slow search | HDD performance issues |
| No results | Index corruption |

---

# Preventive Best Practices

- Keep Windows updated  
- Maintain healthy storage  
- Avoid unnecessary registry modifications  
- Limit indexing to required folders  
- Restart Windows periodically  
- Monitor Event Viewer for search errors  

---

# Escalation Criteria

Escalate when:

- Search fails across multiple devices  
- Index repeatedly corrupts  
- Group Policy conflicts require admin review  
- Windows image cannot be repaired  
- Profile corruption persists  
- Disk reports SMART failures  
- Windows reinstallation is being considered
