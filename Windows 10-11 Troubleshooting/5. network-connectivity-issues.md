# Document 05: Network Connectivity Issues Troubleshooting

## 1. Overview

Network connectivity issues are among the most common incidents handled by IT Support Team. Users depend on reliable network access for:

* Internet browsing
* Microsoft 365 applications
* Cloud services
* VPN connections
* File shares
* Printers
* Business applications
* Video conferencing platforms

A network issue can occur at different layers of the communication process:

* Physical layer (cables, Wi-Fi signals, hardware)
* Network layer (IP addressing, routing)
* Transport layer (TCP/UDP communication)
* Application layer (DNS, browsers, applications)

A structured troubleshooting approach helps identify whether the problem exists on:

* The user's device
* The network adapter
* The local network
* DNS services
* DHCP services
* Firewall/security settings
* Internet Service Provider (ISP)

# 5. Network Troubleshooting Model

IT Support engineers commonly use the **OSI model** to isolate network problems.

```text
+-----------------------------+
| Layer 7 - Application       |
| Browser, Teams, Email       |
+-----------------------------+
| Layer 6 - Presentation      |
| Encryption, Data Format     |
+-----------------------------+
| Layer 5 - Session           |
| Connections                 |
+-----------------------------+
| Layer 4 - Transport         |
| TCP / UDP                   |
+-----------------------------+
| Layer 3 - Network           |
| IP Address, Routing         |
+-----------------------------+
| Layer 2 - Data Link         |
| MAC Address, Switch         |
+-----------------------------+
| Layer 1 - Physical          |
| Cable, Wi-Fi, Hardware      |
+-----------------------------+
```

# 12. Identify Invalid IP Addresses

Problem example:

```
IPv4 Address:

169.254.x.x
```

Meaning:

Windows failed to obtain an IP from DHCP.

---

Solution:

Run:

```cmd
ipconfig /release
```

Then:

```cmd
ipconfig /renew
```

---

## Explanation

### Release

```cmd
ipconfig /release
```

Removes current DHCP address.

---

### Renew

```cmd
ipconfig /renew
```

Requests a new IP address.

---


```cmd

```

Successful:

```

```

---

## Test Internet Connectivity

Run:

```cmd
ping 8.8.8.8
```

Google DNS server.

---

Interpretation:

| Result                  | Meaning            |
| ----------------------- | ------------------ |
| Reply received          | Network working    |
| Request timeout         | Connection problem |
| Destination unreachable | Routing issue      |

# 15. Flush DNS Cache

Old DNS records can cause connection problems.

Run:

```cmd
ipconfig /flushdns
```

---

Explanation:

Deletes stored DNS entries and forces Windows to request updated records.

---

Expected:

```
Successfully flushed the DNS Resolver Cache.
```

---

# 16. Reset TCP/IP Stack

Corrupted networking components can cause connection issues.

Run Command Prompt as Administrator.

Execute:

```cmd
netsh int ip reset
```

---

Explanation:

Resets:

* TCP/IP configuration
* Network registry settings

---

Restart computer after running.

---

# 17. Reset Winsock

Winsock manages Windows network communication.

Run:

```cmd
netsh winsock reset
```

---

Explanation:

Repairs corrupted network socket configuration.

---

Common causes requiring Winsock reset:

* VPN installation
* Malware removal
* Network software conflicts

# 18. Complete Network Reset Script

Create:

```
Scripts/Reset-Network-Stack.ps1
```

---

Script:

```powershell
Write-Host "Resetting Network Configuration..."

ipconfig /release

ipconfig /flushdns

ipconfig /renew

netsh winsock reset

netsh int ip reset

Write-Host "Network Reset Completed"
Write-Host "Restart Computer Required"
```

---

## Script Explanation

### Release IP

```powershell
ipconfig /release
```

Removes existing DHCP assignment.

---

### Flush DNS

```powershell
ipconfig /flushdns
```

Clears cached DNS information.

---

### Winsock Reset

```powershell
netsh winsock reset
```

Repairs Windows network communication.

# 23. Advanced Troubleshooting Using Wireshark

Wireshark captures network packets.

Use cases:

* DNS failures
* TCP connection issues
* Application communication problems

Basic workflow:

```
Start Capture
      |
Select Network Adapter
      |
Generate Problem
      |
Stop Capture
      |
Analyse Packets
```

Common filters:

```
dns
```

Shows DNS traffic.

```
tcp
```

Shows TCP communication.

```
ip.addr == 192.168.1.25
```

Filters traffic from a specific device.

# 26. Lesson Learned

* Always troubleshoot from Layer 1 upward.
* Checking physical connectivity prevents unnecessary advanced troubleshooting.
* IP configuration provides valuable information about DHCP problems.
* DNS issues can appear as internet failures.
* PowerShell enables faster remote diagnostics.
* Proper documentation improves incident resolution time.

---

# 27. Preventive Best Practices

## Users

* Restart network devices when instructed.
* Avoid changing network settings.
* Report exact error messages.
* Maintain updated drivers.

---

## IT Administrators

* Monitor DHCP scopes.
* Maintain DNS availability.
* Update network drivers.
* Document network changes.
* Monitor firewall policies.

---

# 28. Escalation Criteria

Escalate when:

* Multiple users lose connectivity.
* Switch/router failure suspected.
* DHCP server unavailable.
* DNS infrastructure failure occurs.
* ISP outage confirmed.
* Network hardware replacement required.
