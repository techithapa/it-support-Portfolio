# Overview

File Explorer (explorer.exe) is the Windows graphical shell responsible for file browsing, taskbar, desktop, Start menu, and many graphical interface components. When File Explorer repeatedly crashes, freezes, or stops responding, users may be unable to browse files, access network shares, or perform everyday tasks.

Crashes may be caused by corrupted system files, damaged shell extensions, faulty storage devices, problematic Windows updates, corrupted user profiles, or incompatible third-party software.

This guide provides a structured troubleshooting process for diagnosing and resolving File Explorer crashes in Windows 10 and Windows 11.

---

# Symptoms

Users may report:

- File Explorer closes unexpectedly
- Explorer freezes
- Desktop disappears
- Taskbar disappears
- Explorer continually restarts
- Right-click causes Explorer to crash
- Opening folders causes freezing
- Network folders crash Explorer
- High CPU usage by explorer.exe
- Black screen after login

# Step 1 – Identify the Issue

Determine:

- Does Explorer crash immediately?
- Does it crash only when opening folders?
- Does it crash after right-clicking?
- Does it affect all users?
- Did the issue begin after installing software or updates?

Record:

- Error message
- Time of occurrence
- Folder involved
- External devices connected

---

# Step 2 – Restart Windows Explorer

Open Task Manager.

```
Ctrl + Shift + Esc
```

Locate

```
Windows Explorer
```

Select

```
Restart
```

If Explorer is not listed:

```
File

Run new task

explorer.exe
```

---

## Command Prompt

```cmd
taskkill /f /im explorer.exe

start explorer.exe
```

### Explanation

**taskkill**

Terminates the Explorer process.

**start explorer.exe**

Starts a new Windows Explorer process.

# Step 4 – Run System File Checker (SFC)

Open Command Prompt as Administrator.

Run

```cmd
sfc /scannow
```

### Explanation

SFC scans protected Windows system files and automatically repairs corrupted files using the local Windows component store.

Wait for the scan to complete.

Restart the computer.

---

# Step 5 – Repair Windows Image with DISM

If SFC cannot repair all files:

Run

```cmd
DISM /Online /Cleanup-Image /RestoreHealth
```

### Explanation

DISM downloads healthy Windows components from Windows Update and repairs the local Windows image.

Restart the computer after completion.


# Step 8 – Check Third-Party Shell Extensions

Many Explorer crashes originate from third-party context menu extensions.

Examples include:

- Archive software
- Cloud storage applications
- Antivirus products
- PDF software

Disable unnecessary shell extensions using approved administrative tools or uninstall recently installed software.

Restart Explorer after each change.

# Step 10 – Check Disk for Errors

Run

```cmd
chkdsk C: /f
```

If prompted:

```
Schedule on next restart?

Y
```

Restart Windows.

### Explanation

| Command | Purpose |
|----------|----------|
| chkdsk | Checks disk integrity |
| /f | Repairs file system errors |

For bad sectors:

```cmd
chkdsk C: /r
```


# Step 13 – Advanced Troubleshooting

## Create a New User Profile

Create a temporary local administrator account.

Log in.

Verify whether Explorer still crashes.

If the issue disappears, the original profile is likely corrupted.

---

## Check Storage Health

PowerShell

```powershell
Get-PhysicalDisk | Select FriendlyName,HealthStatus,OperationalStatus
```

Healthy example

```
HealthStatus

Healthy
```

---

## Check SMART Status

Command Prompt

```cmd
wmic diskdrive get status
```

Expected output

```
OK
```

---

## Run Windows Memory Diagnostic

Run

```cmd
mdsched.exe
```

Restart the computer.

Allow Windows Memory Diagnostic to complete.

---

## Disable Thumbnail Preview

Open

```
File Explorer Options

View
```

Enable

```
Always show icons, never thumbnails
```

This can prevent Explorer crashes caused by corrupted image previews.

# Verification

Confirm:

- Explorer opens successfully.
- No unexpected crashes occur.
- Desktop loads normally.
- Taskbar remains responsive.
- Right-click menus function correctly.
- Network folders open successfully.
- File copy operations complete normally.
- Event Viewer reports no new Explorer errors.

# Common Error Messages

| Error | Possible Cause |
|---------|----------------|
| Windows Explorer Has Stopped Working | Corrupted system files |
| explorer.exe Application Error | Faulty DLL or shell extension |
| File Explorer Not Responding | Storage or software issue |
| Explorer Keeps Restarting | Corrupted profile |
| Access Violation | Faulty third-party application |
| COM Surrogate Has Stopped Working | Thumbnail or codec issue |
| Explorer Freezes When Right-Clicking | Context menu extension |

# Preventive Best Practices

- Keep Windows updated.
- Regularly check disk health.
- Remove unnecessary shell extensions.
- Keep device drivers current.
- Maintain adequate free disk space.
- Back up important data.
- Monitor storage SMART status.
- Avoid installing unnecessary Explorer extensions.
- Restart Windows periodically.

---

# Escalation Criteria

Escalate when:

- Explorer crashes across multiple devices.
- Storage hardware failure is suspected.
- Memory diagnostics report hardware errors.
- Windows image cannot be repaired.
- Explorer continues crashing after profile recreation.
- Disk reports SMART failures.
- Enterprise software conflicts require vendor support.
- Windows reinstallation is being considered.
