# Overview

Windows Activation ensures the operating system is genuine and properly licensed. Activation failures can prevent users from personalizing Windows, receiving updates, or accessing enterprise resources.

Activation issues may arise from incorrect product keys, corrupted licensing files, KMS server issues, DNS misconfiguration, network restrictions, or damaged system components.

This guide provides a structured troubleshooting process for diagnosing and resolving Windows Activation issues in Windows 10 and Windows 11.

---

# Symptoms

Users may report:

- “Windows is not activated”
- Activation watermark on desktop
- Error codes such as **0xC004F074**, **0xC004F050**, **0x803FA067**
- Unable to personalize desktop
- Activation fails after hardware change
- KMS activation fails on domain devices
- “We can’t activate Windows on this device”
- Activation page stuck or blank

# Common Causes

## Licensing

- Incorrect product key  
- Expired MAK key  
- Invalid OEM key  
- KMS server unreachable  

---

## Network

- DNS misconfiguration  
- Firewall blocking KMS port (TCP 1688)  
- VPN not passing activation traffic  

---

## System

- Corrupted licensing tokens  
- Damaged Windows image  
- Corrupted system files  

---

## Hardware

- Motherboard replacement  
- Major hardware changes  
- BIOS key mismatch

# Step 1 – Identify Activation Status

Open:

```
Settings
System
Activation
```

Check:

- Activation state  
- Error code  
- License type (OEM / Retail / Volume)  

Command Prompt:

```cmd
slmgr /dlv
```

This displays:

- License status  
- KMS server details  
- Activation expiration  
- Error codes  

---

# Step 2 – Run Activation Troubleshooter

Navigate to:

```
Settings
Activation
Troubleshoot
```

The troubleshooter automatically:

- Repairs licensing files  
- Revalidates product key  
- Attempts online activation  

---

# Step 3 – Verify Product Key

Check current key:

```cmd
slmgr /dli
```

Enter new key:

```cmd
slmgr /ipk XXXXX-XXXXX-XXXXX-XXXXX-XXXXX
```

Attempt activation:

```cmd
slmgr /ato
```

---

# Step 4 – Check Activation Services

Ensure required services are running:

```cmd
sc query sppsvc
sc query slsvc
```

Restart services:

```cmd
net stop sppsvc
net start sppsvc
```

---

# Step 5 – Run System File Checker (SFC)

```cmd
sfc /scannow
```

Repairs corrupted system files affecting activation.

Restart Windows.

---

# Step 6 – Repair Windows Image (DISM)

If SFC reports unrepairable files:

```cmd
DISM /Online /Cleanup-Image /RestoreHealth
```

Restart Windows.

---

# Step 7 – Reset Licensing Tokens

```cmd
net stop sppsvc
cd %windir%\System32\spp\store\2.0
ren tokens.dat tokens.old
net start sppsvc
slmgr /ato
```

This rebuilds activation tokens.

---

# Step 8 – Check Network & KMS Connectivity

### Test KMS Port (TCP 1688)

```powershell
Test-NetConnection -ComputerName kms.domain.local -Port 1688
```

### Check DNS SRV Record

```powershell
nslookup -type=SRV _vlmcs._tcp.domain.local
```

Expected:

```
_ vlmcs._tcp.domain.local SRV service location:
```

If missing, KMS activation will fail.

---

# Step 9 – Check DNS SRV Records (Enterprise)

KMS relies on DNS SRV records.

Verify:

```cmd
nslookup -type=SRV _vlmcs._tcp
```

If no records appear:

- DNS replication issue  
- KMS server not publishing SRV records  

---

# Step 10 – Review Event Viewer Logs

Open:

```
eventvwr.msc
```

Navigate to:

```
Applications and Services Logs
Microsoft
Windows
Security-SPP
```

Look for:

- Licensing failures  
- KMS connection errors  
- Token corruption  

---

# Step 11 – PowerShell Activation Diagnostics

Check activation status:

```powershell
Get-ComputerInfo | Select WindowsProductId,WindowsProductName,WindowsActivationStatus
```

Check licensing files:

```powershell
Get-WmiObject -Query "SELECT * FROM SoftwareLicensingProduct"
```

Check KMS client info:

```powershell
cscript slmgr.vbs /dlv
```

# Step 12 – Advanced Troubleshooting

## Hardware Change Reactivation

If motherboard replaced:

```
Settings
Activation
Troubleshoot
I changed hardware on this device
```

Requires Microsoft account binding.

---

## Rebuild Licensing Store

```cmd
sppsvc.exe /rebuild
```

---

## Reinstall Product Key from BIOS (OEM)

```cmd
wmic path softwarelicensingservice get OA3xOriginalProductKey
```

Reapply:

```cmd
slmgr /ipk <key>
slmgr /ato
```

---

## Check KMS Server Health

On KMS server:

```cmd
slmgr /dlv
```

Check:

- Activation count  
- DNS publishing  
- Service status  

---

# Verification

Confirm:

- Windows shows “Activated”  
- No activation watermark  
- Personalization settings unlocked  
- slmgr /dlv shows valid license  
- Event Viewer reports no new activation errors  
- KMS activation renewal succeeds

# Common Error Messages

| Error Code | Meaning |
|------------|---------|
| 0xC004F074 | KMS server unreachable |
| 0xC004F050 | Invalid product key |
| 0x803FA067 | Wrong edition key |
| 0xC004C003 | Key blocked |
| 0xC004E016 | License mismatch |
| 0xC004F009 | Grace period expired |
| 0x8007232B | DNS name does not exist |

# Preventive Best Practices

- Maintain DNS SRV records  
- Ensure KMS server uptime  
- Keep Windows updated  
- Avoid unauthorized key changes  
- Bind device to Microsoft account  
- Maintain accurate system time  
- Monitor activation logs  
- Document hardware changes  

---

# Escalation Criteria

Escalate when:

- Activation fails across multiple devices  
- KMS server is offline or misconfigured  
- DNS SRV records missing enterprise-wide  
- Licensing tokens repeatedly corrupt  
- MAK key activation count exhausted  
- OEM key mismatch after hardware replacement  
- Windows reinstallation is being considered
