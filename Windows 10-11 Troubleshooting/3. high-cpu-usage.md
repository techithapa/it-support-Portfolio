# Document 03: High CPU Usage Troubleshooting

## 1. Overview

High CPU usage occurs when one or more processes consume excessive processor resources, causing the computer to become slow, unresponsive, or unstable.

The CPU (Central Processing Unit) is responsible for executing instructions from:

* Windows operating system processes
* Applications
* Background services
* Security software
* User activities

When CPU usage remains consistently high (90–100%), users may experience:

* Slow application response
* Application freezing
* Delayed keyboard/mouse input
* Fan running continuously
* Increased battery consumption
* System overheating
* Unexpected shutdowns

CPU troubleshooting requires identifying the process causing high usage and determining whether the cause is:

* Application-related
* Windows service-related
* Malware-related
* Hardware-related
* Configuration-related

# 11. PowerShell: Identify Top CPU Processes

Create:

```
Scripts/Top-CPU-Processes.ps1
```

Script:

```powershell
Get-Process |
Sort-Object CPU -Descending |
Select-Object -First 10 `
Name,
CPU,
Id
```

---

## Explanation

### Get-Process

```powershell
Get-Process
```

Displays all running applications and services.

---

### Sort CPU Usage

```powershell
Sort-Object CPU -Descending
```

Sorts processes from highest CPU consumption.

---

### Show Top 10

```powershell
Select-Object -First 10
```

Displays only the highest-consuming processes.

---

Example Output:

```

Teams             540       4210
chrome            320       3020
MsMpEng           200       1150
```

# 12. Real-Time CPU Monitoring Script

Create:

```
Scripts/CPU-Monitoring.ps1
```

Content:

```powershell
while ($true)
{

$cpu = Get-Counter '\Processor(_Total)\% Processor Time'

Write-Host "CPU Usage:"
Write-Host $cpu.CounterSamples.CookedValue "%"

Start-Sleep 5

}
```

---

## Explanation

### Get-Counter

Collects Windows performance data.

---

### Processor Counter

```
\Processor(_Total)\% Processor Time
```

Measures total CPU usage.

---

### Start-Sleep

```
Start-Sleep 5
```

Refreshes every 5 seconds.

---

Example:

```
CPU Usage:
42%

CPU Usage:
88%

CPU Usage:
95%
```

# 23. Lesson Learned

* Task Manager is the first diagnostic tool for performance issues.
* Identifying the responsible process is more effective than restarting the entire system.
* PowerShell provides efficient remote troubleshooting capabilities.
* Security scans are essential when unknown processes consume resources.
* Hardware health directly affects Windows performance.
* Proper documentation improves future incident resolution.

---

# 24. Preventive Best Practices

## Users

* Close unused applications.
* Keep Windows updated.
* Avoid installing unapproved software.
* Restart devices regularly.

---

## IT Administrators

* Monitor endpoint performance using Intune.
* Maintain driver updates.
* Review application deployment impact.
* Use automation scripts for health checks.

---

# 25. Escalation Criteria

Escalate when:

* CPU remains above 90% after troubleshooting.
* Hardware overheating is suspected.
* Malware cannot be removed.
* CPU hardware failure is suspected.
* Multiple devices experience the same issue.
