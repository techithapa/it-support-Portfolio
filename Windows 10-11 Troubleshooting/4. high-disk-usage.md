# Document 04: High Disk Usage (100%) Troubleshooting

## 1. Overview

High disk usage occurs when the storage device (HDD or SSD) is operating at or near maximum capacity. When Task Manager displays **100% Disk Usage**, Windows may become extremely slow because the storage device cannot process read/write requests quickly enough.

Unlike high CPU usage, where the processor is overloaded, high disk usage means applications are waiting for data to be read from or written to storage.

# 5. Common Causes

## Software Causes

### 1. Windows Search Indexing

Windows continuously indexes files to provide faster search results.

Problem:

* Large number of files
* Recently migrated user profile
* New Windows installation

---

### 2. SysMain Service

SysMain (formerly Superfetch) predicts frequently used applications and preloads them.

Problem:

Common on:

* Older HDD systems
* Low-memory devices

---

### 3. Windows Update

During updates Windows may:

* Download packages
* Extract files
* Install patches

---

### 4. Antivirus Scanning

Security software may scan:

* Large files
* Downloads
* System folders

---

## Hardware Causes

### 1. Failing Hard Drive

Symptoms:

* Slow response
* Clicking noises
* Disk errors

---

### 2. Low Free Storage

Windows requires free space for:

* Temporary files
* Virtual memory
* Updates

Recommended:

```text
Minimum: 15-20% free space
```

---

### 3. Insufficient RAM

When RAM is full:

Windows uses:

```text
Page File (Virtual Memory)
```

This increases disk usage.

# 15. Disable SysMain Service

For older HDD systems, SysMain can cause high disk usage.

---

## Check Service

Open:

```cmd
services.msc
```

Find:

```
SysMain
```

---

Stop Service:

1. Right-click SysMain
2. Select Stop

Change:

```
Startup Type:
Disabled
```

---

## PowerShell Method

Run:

```powershell
Stop-Service SysMain

Set-Service SysMain -StartupType Disabled
```

---

Explanation:

First command:

```
Stop-Service
```

Stops current service.

Second command:

```
Set-Service
```

Prevents automatic startup.

---

# 23. Lesson Learned

* Always identify the process causing disk activity before changing services.
* HDD-based systems require different troubleshooting compared to SSD systems.
* Event Viewer is essential for identifying storage failures.
* CHKDSK can repair logical disk issues but cannot fix failing hardware.
* PowerShell automation helps perform health checks efficiently across multiple devices.
* Storage performance directly affects overall Windows experience.

---

# 24. Preventive Best Practices

## Users

* Maintain at least 20% free disk space.
* Avoid storing unnecessary large files locally.
* Restart computers regularly.
* Install applications from trusted sources.

---

## IT Administrators

* Monitor disk health using Intune.
* Replace failing HDDs proactively.
* Standardise SSD deployment.
* Configure storage monitoring alerts.
* Maintain backup procedures.

---

# 25. Escalation Criteria

Escalate when:

* SMART reports disk failure.
* Drive produces unusual noises.
* CHKDSK reports bad sectors repeatedly.
* Multiple users experience the same issue.
* Hardware replacement is required.
