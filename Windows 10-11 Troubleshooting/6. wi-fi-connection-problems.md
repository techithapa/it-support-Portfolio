# Overview

Wireless networking enables users to connect to corporate and home networks without physical Ethernet connections. Wi-Fi connectivity issues can originate from hardware failures, driver corruption, incorrect wireless profiles, DHCP failures, DNS resolution problems, authentication issues, wireless interference, or network infrastructure.

This guide provides a structured troubleshooting process for diagnosing and resolving Wi-Fi connectivity issues on Windows 10 and Windows 11.

---

# Symptoms

Users may report one or more of the following:

- Unable to detect wireless networks
- Unable to connect to Wi-Fi
- Connected but no Internet
- Authentication failures
- Intermittent disconnections
- Slow wireless speed
- Limited connectivity
- Cannot obtain IP address
- VPN disconnects over Wi-Fi
- Microsoft Teams disconnects frequently

# Common Causes

## Hardware

- Wireless adapter failure
- Disabled Wi-Fi adapter
- Loose antenna (desktop)
- Physical wireless switch enabled

---

## Software

- Corrupted wireless driver
- Corrupted TCP/IP stack
- Corrupted Winsock catalog
- Corrupted wireless profile

---

## Network

- DHCP server unavailable
- DNS failure
- Router offline
- Weak wireless signal
- Channel interference

---

## Security

- Incorrect Wi-Fi password
- Expired enterprise certificate
- MAC address filtering
- Firewall restrictions

# Step 1 – Verify Wi-Fi Status

Open:

```
Settings
→ Network & Internet
→ Wi-Fi
```

Ensure:

- Wi-Fi is turned **On**
- Correct wireless adapter is selected

---

## Command Line

Display wireless interfaces.

```cmd
netsh wlan show interfaces
```

### Explanation

Displays:

- Adapter name
- SSID
- Signal strength
- Authentication method
- Radio type
- Receive/Transmit rate

Example:

```text
SSID                 CorporateWiFi

Signal               96%

Authentication       WPA2-Enterprise

State                Connected
```

# Step 2 – Verify Airplane Mode

Open

```
Settings
→ Network & Internet
→ Airplane Mode
```

Ensure:

```
Airplane Mode = Off
```

```powershell
Get-NetAdapter
```

# Step 3 – Forget and Reconnect to Wi-Fi

Navigate to

```
Settings
→ Wi-Fi
→ Manage Known Networks
```

Choose the wireless network.

Select

```
Forget
```

Reconnect by entering the correct password.

This recreates the wireless profile and removes corrupted configuration.

# Step 5 – Verify IP Configuration

Open Command Prompt.

```cmd
ipconfig /all
```

Look for:

- IPv4 Address
- Default Gateway
- DHCP Enabled
- DNS Server

Healthy Example

```text
IPv4 Address

192.168.1.105

Default Gateway

192.168.1.1

DHCP Enabled

Yes
```

Problem Example

```text
169.254.x.x
```

This indicates Windows could not obtain an IP address from DHCP.

---

# Step 6 – Renew DHCP Lease

Release existing IP.

```cmd
ipconfig /release
```

Renew lease.

```cmd
ipconfig /renew
```

### Explanation

**ipconfig /release**

Removes the current DHCP lease.

**ipconfig /renew**

Requests a new IP address from the DHCP server.

---

# Step 7 – Test Network Connectivity

Ping default gateway.

```cmd
ping 192.168.1.1
```

If successful:

```text
Reply from 192.168.1.1
```

Test Internet connectivity.

```cmd
ping 8.8.8.8
```

If this succeeds but websites fail, investigate DNS.

# Step 8 – Verify DNS Resolution

```cmd
nslookup microsoft.com
```

Successful output

```text
Server:

dns.company.local

Address:

10.10.1.5
```

If DNS fails:

Flush DNS cache.

```cmd
ipconfig /flushdns
```

# Step 9 – Restart WLAN AutoConfig Service

Open

```
services.msc
```

Locate

```
WLAN AutoConfig
```

Restart the service.

---

## PowerShell

```powershell
Restart-Service WlanSvc
```

### Explanation

Restarts Windows wireless configuration service.

Without this service Windows cannot automatically connect to wireless networks.

# Step 10 – Reset Network Stack

Open Command Prompt as Administrator.

```cmd
netsh winsock reset

netsh int ip reset

ipconfig /flushdns

ipconfig /release

ipconfig /renew
```

Restart the computer.

### Explanation

| Command | Purpose |
|----------|----------|
| netsh winsock reset | Rebuilds Windows socket catalog |
| netsh int ip reset | Resets TCP/IP configuration |
| ipconfig /flushdns | Clears DNS cache |
| ipconfig /release | Releases DHCP lease |
| ipconfig /renew | Requests new DHCP lease |

Expand

```

```

```

```

- Uninstall device
- Restart computer

# Step 16 – Advanced Troubleshooting

## Signal Strength

Display signal.

```cmd
netsh wlan show interfaces
```

Interpretation

| Signal | Quality |
|----------|----------|
| 90-100% | Excellent |
| 70-89% | Good |
| 50-69% | Fair |
| Below 50% | Poor |

---

## Wireless Profiles

List saved profiles.

```cmd
netsh wlan show profiles
```

Delete corrupted profile.

```cmd
netsh wlan delete profile name="CorporateWiFi"
```

Reconnect to recreate the profile.

---

## Check Network Adapter Power Saving

Open

```
Device Manager

Network Adapter

Properties

Power Management
```

Disable

```
Allow the computer to turn off this device to save power
```

This prevents unexpected wireless disconnects.

# Verification

Confirm the following:

- Connected to correct SSID
- Valid IPv4 address assigned
- Gateway reachable
- Internet accessible
- DNS resolving correctly
- VPN connects successfully
- Microsoft Teams remains connected
- No authentication errors
- WLAN report shows successful connection

# Common Error Messages

| Error | Possible Cause |
|---------|----------------|
| Can't connect to this network | Incorrect password or corrupted profile |
| No Internet, Secured | DHCP or DNS issue |
| Limited Connectivity | Router or DHCP problem |
| Network Security Key isn't Correct | Incorrect password |
| Windows Couldn't Connect | Driver or profile corruption |
| No Wi-Fi Networks Found | Disabled adapter or driver issue |

# Preventive Best Practices

- Keep wireless drivers updated.
- Use WPA2 or WPA3 security.
- Maintain current Windows updates.
- Remove unused wireless profiles.
- Regularly restart wireless infrastructure.
- Avoid crowded wireless channels.
- Monitor signal strength in office environments.
- Keep firmware updated on wireless access points.

---

# Escalation Criteria

Escalate when:

- Multiple users cannot connect simultaneously.
- Wireless access points are offline.
- DHCP server is unavailable.
- Enterprise authentication servers fail.
- Certificate authentication fails.
- Wireless controller reports hardware faults.
- Driver replacement does not resolve the issue.
- Hardware replacement is required.
